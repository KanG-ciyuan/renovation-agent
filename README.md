# 装修咨询智能客服 Agent Demo

这是一个可本地运行、可继续接入扣子与飞书的最小可交付案例，用来证明你具备以下能力：

- 能把 AI/Agent 思路落成可演示产品
- 能围绕真实商家场景设计自动化流程
- 能写出并跑通一段代码
- 能把客户咨询转成结构化线索，方便同步到飞书或 CRM

## 这个案例解决什么问题

面向装修公司或家装工作室，用户常见咨询包括：

- 装修价格
- 装修风格建议
- 工期安排
- 是否支持先看效果图
- 是否可以预约量房

这个 demo 会把用户咨询自动做成：

- 客服回复建议
- 客户意图识别
- 预算/面积/风格/房型等信息提取
- 线索意向分级
- 销售跟进建议
- 飞书同步载荷预览

## 你可以怎么对外描述

你可以把它说成：

> 我做了一个装修公司智能客服与线索跟进 Agent。用户咨询后，系统会自动识别客户意图、提取面积预算风格等关键信息，生成客服回复，并输出结构化线索，后续可以同步到飞书表格或 CRM 给销售跟进。这个项目我自己完成了从场景设计、Agent 流程拆解到代码落地的全过程，目前已经可以本地演示。

## 运行方式

系统默认不依赖第三方包，直接运行：

```bash
node app.js
```

启动后访问：

`http://localhost:3000`

## 当前版本的回复逻辑是什么

当前默认是“本地规则版”：

- 代码会根据关键词判断客户意图
- 代码会提取面积、预算、风格、房型、地区
- 代码会按规则生成首轮客服回复

这意味着现在的回复不是大模型实时生成，而是你为了做演示先写好的业务规则。

## 怎么切到扣子智能体

项目已经预留了扣子接入位。你只需要准备：

- `COZE_API_TOKEN`
- `COZE_BOT_ID`
- 可选的 `COZE_BASE_URL`

然后把环境变量设置好，再启动：

```bash
AI_MODE=coze COZE_API_TOKEN=你的token COZE_BOT_ID=你的bot_id node app.js
```

接入后，系统会：

1. 先保留本地提取出的客户画像
2. 再把客户原始消息发给扣子
3. 用扣子的回复替换本地固定话术

这样你就得到一个“规则提取 + 智能体回复”的版本。

## 怎么接到飞书

项目也预留了飞书 webhook 接入位。你需要准备一个飞书群机器人 webhook 地址：

- `FEISHU_WEBHOOK_URL`

启动方式示例：

```bash
FEISHU_WEBHOOK_URL='你的飞书webhook' node app.js
```

这样每次分析后，项目都会把结构化线索自动推送到飞书群。

## 推荐你现在的成品路线

按这个顺序最稳：

1. 先用当前本地规则版跑通网页
2. 再接入扣子，让回复更像真实客服
3. 再接飞书，把线索自动通知到群里

## 演示建议

你可以现场输入这些内容：

1. `我在红谷滩有一套89平二手房，想做奶油风，预算大概12万，多久可以开工？`
2. `旧房翻新多少钱一平？可以先出效果图吗？`
3. `这周六方便上门量房吗？我家在南昌红谷滩。`

系统会自动输出：

- 客服回复
- Agent 分析过程
- 结构化客户档案
- 跟进建议
- 可同步到飞书的 JSON
